About

The feature of Vanity Imports (or remote import paths, as they are called in the reference) have been implemented for a while, but does not seem to be used that often. I assume, because it makes you maintain an HTML header section. This is, what ImportAlias will do for you.

Getting started

I tried to keep it as simple as possible. You don't need to remember any credentials, I only offer 3rd party login. Right now I support GitHub and Google. Just click the Login button in the top right corner.

Before you can add an alias route, you need to claim a domain. Right now, a domain can be managed by one person (one account) only. The architecture has been designed to support delegates, but the functionality has not been exposed just yet. To verify that you are actually the owner of the domain you are about to claim, a DNS request is made to check, if said domain contains a TXT record containung your UID. The DNS request is made against Google’s DNS and not resolved manually so give the record some time to propagate.

Example: Let’s assume you want to claim the domain surmair.de. Go to your DNS provider and make an entry for _importalias.surmair.de and set the content to your UID. You can find your UID (as well as your personal API key) in the dropdown in the navigation bar at the top.

Now that you have your domain you can add aliases. Each alias has some information saved about them:

Alias
The path which should be aliased for a remote repository. It may contain multiples slashes.
Repository
Type and URL of said repository. At the moment Git, Mercurial and Bazaar are supported by go get.
Forward URL
The URL to forward to when it is accessed by something else than go get (e.g. a browser).
