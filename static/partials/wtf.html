About

Remote import paths (as they are called in the reference) have been implemented for a while, but do not seem to be used that often. I assume this is because you need to maintain an HTML header section. This is what ImportAlias will do for you.

Getting started

I tried to keep the overhead as small as possible. You don't need to remember any credentials, I only offer 3rd party login. Right now I support GitHub and Google. Just click the Login button in the top right corner.

Before you can add an alias route, you need to claim a domain. Right now a domain can be managed by one person (one account) only. The architecture has been designed to support delegates, but the functionality has not been exposed just yet.

To verify that you are actually the owner of the domain you are about to claim, a DNS request is made to check if said domain contains a TXT record containung your UID. The DNS request is made against Google’s DNS and not resolved manually so give the record some time to propagate.

To make ImportAlias actually work, another DNS record is needed: A CNAME record which points to importalias.surmair.de. This way, all requests sent to that domain will be handled by the ImportAlias app.

Example: Let’s assume you want to claim the domain go.example.com. Go to your DNS provider and make an entry for _importalias.go.example.com and set the content to your UID. You can find your UID (as well as your personal API key) in the dropdown in the navigation bar at the top. Also add a CNAME record for go.example.com and let it point to importalias.surmair.de.

Now that you have your domain set up you can add aliases. For each alias, the following information is required:

Alias
The path which should be aliased for a remote repository. It may contain multiples slashes.
Repository
Type and URL of said repository. At the moment Git, Mercurial and Bazaar are supported by go get.
Forward URL
The URL to forward to when it is accessed by something else than go get (e.g. a browser).

Alpha

Important: I give this service the “alpha stage” mark. Nothing has been tested in the wild and I will not give any guarantees. If you rely on your Go imports and need them to work 24/7, do not use this service just yet. On the contrary: Try to break it, do weird stuff, use the API.
